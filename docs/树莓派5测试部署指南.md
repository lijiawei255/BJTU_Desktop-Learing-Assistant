## 1. 准备工作：硬件清单

### 1.1 核心设备

|------|-------------|---------|------|
| PCA9685舵机驱动板 | `servo_controller.py` | I2C (0x40) | 驱动4路舵机 |

| 自复开关(按钮) | `gpio_button.py` | GPIO 27 | 休眠/唤醒/重启 |

## 2. 树莓派系统配置

> 以下命令均在树莓派上执行（通过SSH或VNC终端）。

### 2.1 系统更新

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git vim tmux i2c-tools alsa-utils usbutils
```

#### 检查USB音频内核支持

USB声卡依赖内核的 `snd_usb_audio` 模块，需确认内核已编译该驱动：

```bash
# 检查模块是否已加载
lsmod | grep snd_usb_audio

# 检查内核是否编译了该模块（即使未加载）
modinfo snd_usb_audio
```

| 情况 | 处理 |
|------|------|
| `modinfo` 有输出 | 模块可用，执行 `sudo modprobe snd_usb_audio` 加载 |
| `modinfo` 无输出 | 内核裁剪了该驱动，需重新编译内核（RPi OS 默认已编译，极少遇到） |

若需编译内核，在 `.config` 中开启：
```
CONFIG_SND_USB_AUDIO=m
CONFIG_SND_USB_UA101=y
CONFIG_SND_USB_CAIAQ=y
```

### 2.2 启用硬件接口

```bash
sudo raspi-config
```

在图形界面中启用以下接口：

| 菜单路径 | 选项 | 说明 |
|----------|------|------|
| Interface Options → I2C | **Enable** | PCA9685 + VL53L0X通信 |
| Interface Options → SPI | **Enable** | 预留，当前未用 |
| Interface Options → Camera | **Enable** | CSI摄像头 |
| Interface Options → Remote GPIO | **Enable** | 允许远程GPIO访问 |

完成后重启：

```bash
sudo reboot
```

### 2.3 用户权限

将当前用户加入硬件访问组：

```bash
sudo usermod -a -G gpio,i2c,spi,video,audio $USER
```

> **注意**：执行后需重新登录才能生效。可运行 `groups` 确认当前用户所属组。

### 2.4 验证I2C总线

重启后，检查I2C设备：

```bash
# 扫描I2C总线1（树莓派5主I2C接口）
sudo i2cdetect -y 1
```

未接设备时，输出为空网格（全 `--`）。接线后会显示设备地址（如 PCA9685 的 `0x40`、VL53L0X 的 `0x29`）。

### 2.5 音频降噪配置（USB声卡噪音修复）

树莓派5使用 USB 声卡时，若录制音频有明显底噪，在 `/boot/firmware/config.txt`（或 `/boot/config.txt`）末尾添加：

```bash
sudo nano /boot/firmware/config.txt
```

末尾添加一行：

```
audio_pwm_mode = 2
```

> `audio_pwm_mode = 2` 将音频 PWM 改为更优模式，可显著降低底噪。
> 修改后需重启：`sudo reboot`

### 2.6 创建项目目录

```bash
mkdir -p /home/pi/amiya
```

### 4.6 音频设备配置（UACDemoV1.0 USB声卡）

> 树莓派5使用的音频硬件为 **UACDemoV1.0 USB声卡**（集成麦克风+音箱接口），下面按 ALSA 层 → PyAudio 层的顺序配置。

#### 4.6.1 ALSA系统层：确认USB声卡被识别

```bash
# 安装 ALSA 工具（通常已预装）
sudo apt install -y alsa-base alsa-utils

# 插入USB声卡前，查看已有USB设备
lsusb

# 插入USB声卡后再次查看，对比新增设备
lsusb
# 预期看到：UACDemoV1.0 或类似的 USB Audio 设备
```

列出录音和播放设备：

```bash
arecord -l   # 列出所有录音设备
aplay -l     # 列出所有播放设备
```

预期输出示例：

```
**** List of CAPTURE Hardware Devices ****
card 1: UACDemoV1.0 [UACDemoV1.0], device 0: USB Audio [USB Audio]
  Subdevices: 1/1
  Subdevice #0: subdevice #0
```

> 在树莓派5上，板载音频通常为 card 0，USB声卡为 card 1。
> ALSA设备标识为 `plughw:1,0`（card 1, device 0）。

#### 4.6.2 音量调节（关键步骤）

USB声卡默认音量可能很低，必须手动调高：

```bash
alsamixer
```

操作步骤：
1. 按 **F6** → 用方向键选择 **UACDemoV1.0** → 回车
2. 按 **F5** 显示所有控件（录音+播放）
3. 用 **↑/↓** 方向键调高：
   - **PCM**（播放音量）：调到 80~90
   - **CAPTURE** 或 **MIC**（录音增益）：调到 70~80
4. 如果某项显示 **MM**（静音），选中后按 **M** 取消静音
5. 按 **ESC** 退出

保存音量设置（每次重启音量会被初始化，必须持久化）：

```bash
# 修复权限（默认可能为只读）
sudo chmod 664 /var/lib/alsa/asound.state
# 保存当前音量到系统默认配置文件
sudo alsactl store
```

> **注意**：每次手动调整音量后，需要重新执行 `sudo alsactl store` 才能持久化。

#### 4.6.3 ALSA层回路测试

先不依赖Python，用系统命令验证录音和播放：

```bash
# 首先确认 USB 声卡被系统枚举
lsusb | grep -i audio
aplay -l
# 预期看到 USB Audio 相关的 card X 条目

# 可选：播放测试音确认音箱正常（X=USB声卡的 card 编号）
speaker-test -c 2 -D plughw:X,0

# 录制5秒测试音频（根据 arecord -l 的输出调整 card,device）
arecord -D plughw:1,0 -f S16_LE -r 16000 -d 5 -t wav test.wav

# 播放刚才的录音
aplay -D plughw:1,0 -f S16_LE -r 16000 -c 1 test.wav
```

参数说明：
| 参数 | 含义 | 本项目值 |
|------|------|---------|
| `-D plughw:1,0` | ALSA设备（card 1, device 0） | 根据 `arecord -l` 调整 |
| `-f S16_LE` | 16位有符号小端序 | 与项目 `paInt16` 一致 |
| `-r 16000` | 16kHz采样率 | 与项目 `audio.sample_rate` 一致 |
| `-c 1` | 单声道 | 与项目一致 |

> **常见问题**：如果录音音量仍然很小，回到 `alsamixer` 继续调高 Capture/MIC 增益。

#### 4.6.4 PyAudio层：找到设备索引

运行项目自带的音频测试脚本：

```bash
cd /home/pi/amiya
conda activate amiya
python -m tests.test_real_audio
```

输出会列出所有 PyAudio 检测到的输入/输出设备及索引。

> ALSA 的 `card 1` **不一定等于** PyAudio 的 `index=1`。PyAudio 按自己的顺序枚举设备，需以 `test_real_audio` 的输出为准。

#### 4.6.5 固定设备索引

根据上面 `test_real_audio` 的输出，在 `.env` 中设置正确的索引：

```env
# USB声卡（UACDemoV1.0）的 PyAudio 设备索引
AUDIO_INPUT_DEVICE_INDEX=1
AUDIO_OUTPUT_DEVICE_INDEX=1
```

> **注意**：输入和输出可能使用不同索引号（取决于PyAudio枚举顺序），以实际输出为准。
> 树莓派5上 USB 声卡的典型值为 index 1 或 2。

## 5. 逐设备测试

> **核心策略**：利用 `data/config.json` 中各设备的独立 `mock` 开关，按电气依赖从低到高、风险从少到多的顺序逐步启用真实硬件。每步只改一个开关，出了问题能立刻定位原因。

### Mock开关原理

在 `data/config.json` 的 `mock` 段中：

```json
"mock": {
    "enabled": true,    // ← 全局总开关。false时所有设备走真实驱动
    "audio": true,      // ← 每设备独立开关。true=mock模式 false=真实硬件
    "camera": true,
    "servo": true,
    "tof": true,
    "ir": true,
    "led": true,
    "button": true
}
```

两个条件同时为 `true` 才走 Mock：`mock.enabled && mock.音频/摄像头/舵机/...`。

### 5.2 真实音频测试（`mock.audio = false`）

**需要硬件**：USB声卡+麦克风+音箱。

修改 `data/config.json`：

```json
"mock": {
    "enabled": true,
    "audio": false,   // ← 只改这一行
    "camera": true,
    "servo": true,
    ...
}
```

**测试步骤**：

1. 确认音频设备连接正常：
   ```bash
   arecord -l   # 列出录音设备
   aplay -l     # 列出播放设备
   ```

2. 运行音频回环测试（可选）：
   ```bash
   python -m tests.test_real_audio
   ```

3. 启动主程序，进行语音对话：
   ```bash
   python -m src.main
   ```

4. 对着麦克风说「阿米娅」唤醒词，验证以下环节：
   - VAD 能检测到语音
   - ASR 正确识别唤醒词
   - TTS 播放 "嗯，我在听，请说。"
   - 多说一句话，验证多轮对话正常

**常见问题排查**：

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| `No Default Input Device` | PyAudio找不到麦克风 | `arecord -l` 确认设备存在；在 `.env` 中设置 `AUDIO_INPUT_DEVICE_INDEX` |
| 录音无声音 | 麦克风未正确连接 | 用 `arecord test.wav` 录制测试文件 |
| ASR返回空/NULL | 网络不通或API Key无效 | `curl https://dashscope.aliyuncs.com` 测试连通性 |
| TTS无声音 | 音箱未连接或音量太小 | `alsamixer` 调整音量；`aplay /usr/share/sounds/alsa/Front_Center.wav` 测试 |
| 声音像"快进" | 采样率不匹配 | 确认 `audio.sample_rate=16000` |
| VAD不检测语音 | 麦克风增益太低 | 调整 `alsamixer` 中的 Capture 增益 |
| 录音有明显底噪 | 树莓派PWM音频干扰 | 在 `/boot/firmware/config.txt` 末尾加 `audio_pwm_mode = 2`，重启 |
| 重启后音量恢复默认 | 未持久化音量设置 | `sudo chmod 664 /var/lib/alsa/asound.state && sudo alsactl store` |

### 5.3 LED测试（`mock.led = false`）

**目的**：验证GPIO输出——最简单的硬件驱动测试。

**需要硬件**：RGB LED模块（共阴），3个GPIO引脚 + GND。

**接线**：

| LED引脚 | 树莓派GPIO (BCM) | 物理引脚 |
|---------|-----------------|---------|
| R（红色） | GPIO 23 | Pin 16 |
| G（绿色） | GPIO 24 | Pin 18 |
| B（蓝色） | GPIO 25 | Pin 22 |
| GND | GND | Pin 20 |

> 建议在每个颜色引脚串联 220Ω~470Ω 限流电阻。

修改 `data/config.json`：

```json
"mock": {
    "enabled": true,
    "audio": false,
    "led": false,      // ← 启用真实LED
    ...
}
```

**测试方法**：

```bash
# 快速测试：直接通过Python操作
python -c "
from src.devices import get_led
led = get_led()
led.set_color('red', 'solid')
led.set_color('green', 'blink')
led.set_color('blue', 'breath')
led.off()
print('LED test passed!')
"
```

**预期**：LED分别显示红色常亮→绿色闪烁→蓝色呼吸→熄灭。

**常见问题**：

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| LED不亮 | 接线错误 | 用万用表确认RGB三个引脚连接正确 |

### 5.7 舵机测试（`mock.servo = false`）

> - PCA9685与树莓派必须**共地**（GND相连）。
> - 接线前用 `sudo i2cdetect -y 1` 确认 `0x40` 可见。

## 附录E：PC→树莓派文件传输方法

